Kirki features

Why should you use it?

We recently started rebuilding the Shoestrap theme and needed to add some custom controls (like multi-check checkboxes, sliders using the jQuery SliderUI, buttonsets, layout selection etc). Kirki is the result of these efforts. If you want to see a demo of Kirki, we would advise you to download and install the (still incomplete - under heavy development) new Shoestrap theme from github.

Easy configuration and initialization

There are 2 ways to install the plugin:

  • Install as a plugin:

    If you install it as a plugin then you can go ahead with its configuration as described below.

  • Embed in your theme

    If you decide to embed Kirki in your own theme then you will have to follow the steps below:

    1. Copy the plugin folder in your theme (for example in {theme_folder}/kirki).
    2. Include the main plugin file in your theme's functions.php file: 
      include_once( dirname( __FILE__ ) . '/kirki/kirki.php' );
    3. Add a filter to change the URL that will be used for the plugin's assets. This can be done from the configuration below (see lines 32-35).

After you install the plugin (or embed it in your theme), you can customize its look and make it match your theme's branding and feel with just a few lines!

Additional Options

In the example above, we created a simple text field that has a label and the text input itself. In some cases, you might want to add some more info to that field and provide additional information so that your users know exactly what your field does and how to use it.

For that reason, we've added some additional arguments you can use: subtitle and description. Example:

Adding a Buttonset control:

	'type'     => 'radio',
	'mode'     => 'buttonset',
	'setting'  => 'my_setting',

	'section'  => 'my_section',
	'default'  => 'option_1',
	'priority' => 1,
	'choices'  => array(
		'option_1' => __( 'Option 1', 'textdomain' ),

Adding an Background control:

The background control is a bit more complicated that others: 
$controls[] = array(
	'type'         => 'background',
	'setting'      => 'my_setting',
	'label'        => __( 'My Setting', 'textdomain' ),
	'description'  =>   __( 'Background Color', 'shoestrap' ),
	'section'      => 'my_section',
	'default'      => array(
		'color'    => '#ffffff',
		'image'    => null,
		'repeat'   => 'repeat',
		'size'     => 'inherit',
		'attach'   => 'inherit',
		'position' => 'left-top',
		'opacity'  => 100,
	),
	'priority' => 3,
	'output' => 'body',
);
What this actually does, is create 7 other settings and controls:
  • my_setting_color (color)
  • my_setting_image (image upload)
  • my_setting_repeat (dropdown)
  • my_setting_size (buttonset)
  • my_setting_attach (buttonset)
  • my_setting_position (dropdown)
  • my_setting_opacity(slider)

The my_setting option is never created, and only serves as a prefix for the other options.

if you want to disable opacity for your control, then you can simply set 'opacity' => false in the defaults section.

Since these are essentially 7 controls, each contron has its own priority. So when setting the prioritis for your controls, please make sure you leave enough room for all 7 of them.

The background control also allows you to define if you want the CSS to be automatically generated and applied for you. If you want to enable this feature, then you can simply set 'output' to the element whose background you want to control (in our above example, the <body> of our page).

Please note that the generated CSS is then added to your site using the wp_add_inline_style() function, so in order for this to work you must first specify the stylesheet_id in the plugin's configuration options.

Field "required" argument

Sometimes it is useful to hide a few of our fields until the user selects an option that "activates" them.

For example you might have sliders that control the width and height of your post featured images, but you don't want to show these settings until the user selects that they actually want to display featured images on their posts.

For that we've introduced the required argument.

Notice: The element that triggers the fields to be hidden or shown at the moment can only be a buttonset field.

example:

$controls[] = array(
	'type'     => 'radio',
	'mode'     => 'buttonset',
	'setting'  => 'parent_setting',
	'label'    => __( 'My Setting', 'textdomain' ),
	'section'  => 'my_section',
	'default'  => 'option_1',
	'priority' => 1,
	'choices'  => array(
		'option_1' => __( 'Option 1', 'textdomain' ),
		'option_2' => __( 'Option 2', 'textdomain' ),
		'option_3' => __( 'Option 3', 'textdomain' ),
		'option_4' => __( 'Option 4', 'textdomain' ),
	),
);

$controls[] = array(
	'type'     => 'textarea',
	'setting'  => 'child_setting',
	'label'    => __( 'My Setting', 'textdomain' ),
	'section'  => 'my_section',
	'default'  => __( 'Default text', 'textdomain' ),
	'priority' => 1,
	'required' => array( 'parent_setting' => 'option_2' )
);

In the above example, the child_setting will be hidden unless the parent_setting is set to option_2.
